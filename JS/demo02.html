JavaScript Data types & Variables!

JS variables fundamentals
  • variables are named memory locations to hold the data.
  • variables are auto-declared upon assignment.
  • data types are inferred (auto-detected).
  • data types: number, boolean, string, object.
  • strings can be enclosed in '..', "..", or `..`.
  • templated strings to print var values: `var=${varname}`
JS var, let, and const keywords
  • variables can be declared explicitly using "var" keyword.
  • variables created without "var" are global -- called as implicit variable declaration.
  • variables created with "var" - have function scope.
  • variables can be declared explicitly using "let" keyword.
  • constant variables can be declared explicitly using "const" keyword.
  • variables created with "let" & "const" - have block scope.
  • modern JS prefer "let" & "const" over "var" keyword.
  • "var" variables can be redeclared in same scope, but "let" & "const" can't.
Note: scopes will be covered later.

JS operators
  • == operator compares the contents (but not the data type).
  • === operator compares the contents as well as data types.
  • similarly != compare contents, while !== compraes contents + data types.
  • Logical && returns first falsy value (this may be false, 0, empty string, or null)
  • Logical || returns first truthy value (this may be true, non-zero number, non-empty string, or non-null object)
